## 六、对话窗口交接规范

### 触发条件
当与 AI 的对话窗口接近上限时（通常在长时间连续开发对话后），**必须提前**执行以下交接流程，不要等到窗口真正满了才处理。

### 交接文档模板
每次交接时，在 `docs\handover\` 目录下创建文件：
`YYYY-MM-DD_HH-MM_开发交接.md`

**交接文档必须包含**：

```markdown
# 开发交接文档
日期：
当前阶段：第X阶段 - [阶段名称]

## 1. 当前进度
- 已完成的功能模块
- 正在进行中的任务（精确到哪个文件、哪个函数）

## 2. 文件状态
- 最后修改的文件列表
- 每个文件当前的状态说明

## 3. 待解决问题
- 遗留的 Bug 或未完成的逻辑

## 4. 下一步任务
- 新窗口第一件事是什么
- 具体到文件名和函数名

## 5. 新窗口启动指令
新对话开始时，把以下内容发给 Claude：

> 我正在开发 The Syntax Atelier（句法工坊），
> 请先阅读以下交接文档，然后继续开发工作：
> [粘贴本交接文档全文]
```

### 重要原则
- **提前交接**：对话窗口还剩 20% 容量时就开始写交接文档，不要拖到最后
- **状态完整**：交接文档必须让新窗口的 Claude 无需看历史对话就能接手工作
- **代码优先**：交接前确保当前改动已保存，关键代码已注释清楚
