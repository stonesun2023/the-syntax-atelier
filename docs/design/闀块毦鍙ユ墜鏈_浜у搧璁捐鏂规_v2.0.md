# The Syntax Atelier · 句法工坊
## 产品设计方案 v2.0
**日期**：2026-03-01  
**开发模式**：Vibe Coding（直觉驱动型探索）  
**定性**：开源 · 学生自用 · AI 驱动 · 数据归用户

---

> **产品核心信条**
> 翻译不是搬运砖头（单词），而是搬运蓝图（逻辑）。
> 本工具的使命不是给你一个答案，而是教你看懂一个句子。

---

## 一、产品定位

### 核心一句话
一款帮助任何想学好英语的人，通过"逻辑解构"真正读懂英文长难句的 AI 交互工具。

### 用户画像
- **核心用户**：备考四六级、考研的大学生
- **扩展用户**：所有希望提升英文阅读与理解能力的中文母语者
- **使用动机**：遇到读不懂的长句时主动解析；或进入挑战模式系统性训练

### 与通用 AI 的本质区别
通用 AI（如 ChatGPT）给你一个翻译结果，但不告诉你为什么。本工具有一套**专属方法论**，把解构过程完整呈现，让用户学会思维方式，而不是依赖答案。

### 不做什么（MVP 明确排除）
- 中译英方向
- 账号注册 / 登录系统（数据存本地）
- 服务端数据库
- 预设静态题库（全部由 AI 动态生成）
- 其他外语（日语、法语等）
- 移动端 App（先做网页版）

## 二、核心方法论（产品灵魂）

### 2.1 四步解构算法 (The 4-Stage Engine)

| 步骤 | 名称 | 动作 | 目标 |
|------|------|------|------|
| 1 | 信号探测 Signal Detection | 识别 which/that/because 等连接词，锁定介词短语与非谓语动词 | 找到句子的"关节点" |
| 2 | 骨架提取 Skeleton Extraction | 屏蔽所有修饰成分（定语、状语、补语），像剪枝一样剔除 | 锁定核心 S（主）+ V（谓）+ O（宾） |
| 3 | 逻辑分层 Layering | 划分：主脉（主句核心）/ 支脉（定语从句）/ 背景（状语）/ 嵌套（从句套从句） | 建立句子的层级树 |
| 4 | 语序重排 Re-ordering | 遵循中文叙事习惯：状语前置、被动转主动、结果后置 | 从"外语脑"切换到"中文脑" |

**专家口诀**（对应三步操作）：
- **剪枝（Peel）**：先找骨架，再填肉
- **断句（Slice）**：长句化竹，一节一节读
- **调序（Flip）**：英语先果，中文先因

### 2.2 三阶译文标准

| 级别 | 名称 | 面向用户 | 教学重点 | 译文特征 |
|------|------|----------|----------|----------|
| Level 1 | 骨架译 Foundation | 语法基础薄弱，翻译"不像人话" | 找准主干，能把"谁做了什么"找出来就成功 | 直白平铺，主谓宾齐全，略显生硬 |
| Level 2 | 逻辑译 Intermediate | 能看懂结构，但处理语序差异困难 | 逻辑断句与重组，强调"前因后果"的中文思维 | 语序通顺，符合中文阅读直觉，无翻译腔 |
| Level 3 | 神韵译 Professional | 已掌握结构转化，追求信达雅 | 意象捕捉与脱壳，翻译"背后的含义"而非单词本身 | 词炼字精，具备语境深度，有文学张力 |

### 2.3 万能转化套路（Universal Patterns）

这是教学的核心工具，也是挑战模式的出题素材库：

| 英文结构 | 中文转化方式 | 例示 |
|----------|-------------|------|
| If / When / Without... | 先说"如果不/当..."，再说结果 | Without X, Y will... → 若缺乏X，则Y将... |
| A which/that is B | 拆成两句："这就是A，它很B" | The risk, which is growing... → 这个风险正在增长... |
| A shifts from B to C | "重心变了：从B变成了C" | Risk shifts from bugs to debt → 风险从缺陷转向债务 |
| Render / Make A + Adj | "导致A变得..." | Render the system inscrutable → 导致系统变得晦涩难懂 |
| Rather than Y, X... | 先立X，再破Y | Acting as curators rather than authors → 成了管理员，而非创造者 |
| Only by doing... | 状语前置 | Only by working together can we... → 只有通力合作，我们才能... |
| Had + 主语 + 过去分词 | 虚拟倒装还原 | Had they known... → 如果他们当时知道... |

### 2.4 学生自检清单（每次解构后使用）

- 剪枝了吗？能在 3 秒内找到主干 S+V+O？
- 断句了吗？长达 20 词的句子拆成了符合呼吸频率的短句？
- 调序了吗？状语（时间/条件/地点）挪到了结论前面？
- 说人话了吗？读一遍译文，中文母语者能听懂吗？
- 大词转化了吗？术语是否转化成了直白的意象？

## 三、功能模块详细设计

### 模式一：解构模式（学）

**用户流程**：输入任意英文长句 → AI 实时解构 → 可视化呈现结果

**输入方式**：
- 手动粘贴（真题、新闻、博客、任意来源均可）
- 历史句子回顾（本地存储，可重新解构）

**输出内容**：

**① 骨架高亮层**
原句以彩色色块呈现，支持"全句模式"与"骨架模式"动态切换：
- 主语：蓝色
- 谓语：绿色
- 宾语/补语：橙色
- 修饰成分（定语/状语）：灰色
- 连接词/信号词：紫色下划线

**② 逻辑色块层**
每个成分附带功能标签，点击后弹出"语法气泡"，用极简语言说明原理（拒绝陈腐术语）：
- 主脉（Main）：主句的核心动作
- 背景（Context）：时间/地点/条件状语
- 支脉（Modifier）：定语从句，修饰名词
- 嵌套（Nested）：从句中的从句

**③ 三阶译文层**
骨架译 → 逻辑译 → 神韵译，默认折叠，用户逐级展开（防止直接看答案）。
每阶附带一句说明，解释这一级的翻译策略是什么。

**④ 专家提示（Expert Alert）**
遇到特殊结构时自动触发，例如：
- 倒装句：「检测到 Had 前置，这是虚拟语气倒装结构，真实逻辑是：如果他们当时知道…」
- 虚拟语气：「这里表达的是与过去事实相反的假设，不是真实发生的事」
- 插入语：「括号内成分与主句逻辑无关，可暂时忽略」

**⑤ 万能套路匹配**
系统自动识别句子中出现的万能转化套路，并在解析结果底部显示对应的规律提示。

### 模式二：挑战模式（练）

**核心逻辑**：AI 根据用户在解构模式中输入过的句子，自动出题。用户学过的内容，就是出题的素材。

**题型设计**（三个级别，对应方法论三阶）：

**L1 剥离术（基础关）**
- 题目：给出一个长句，从四个选项中找出正确的主干 S+V+O
- 考查：骨架提取能力
- 例：「In an era dominated by digital communication, the art of handwriting is fading.」→ 哪个是正确主干？

**L2 语序重组（进阶关）**
- 题目：给出四个中文译文，选出语序最符合中文逻辑的版本
- 考查：语序重排能力，状语是否前置，逻辑是否顺畅

**L3 神韵对比（专家关）**
- 题目：给出四个译文，选出最准确传达意境、最具文学张力的版本
- 考查：意象捕捉与脱壳能力
- 例：「The silence was palpable.」→ 「屋子里的寂静仿佛凝固了，触手可及」vs「房间里的沉默是可以被摸到的东西」

**心理战术题（终极关）**
- 题目：面对特定复杂结构（如双重否定+插入语），专家的第一步动作是什么？
- 考查：解构思维与方法论的内化程度

**交互设计**（参考 Gemini 案例）：
- 顶部进度条：当前题号 / 总题数，实时显示对错计数（红色×，绿色✓）
- 单题单页，选择后立即呈现：正确答案高亮（绿色）+ 原理解释
- 跳过题目也显示正确答案（标注 skipped）
- 完成页面展示：总分 / 准确率 / 答对 / 答错 / 跳过
- 「分析我的表现」按钮：AI 总结本次挑战的强项与盲点

### 模式三：学习档案（追踪）

**数据来源**：用户在挑战模式中的所有答题记录，存储于本地 IndexedDB

**功能模块**：

**① 能力雷达图**
五个维度可视化：骨架提取 / 逻辑断句 / 语序重排 / 意象捕捉 / 边界防御
直观显示哪个维度是强项，哪个是盲点

**② 盲点标注**
系统自动统计哪类语法结构反复出错（如：定语从句正确率 62%，虚拟语气正确率 41%）
并推送针对性的复习建议

**③ 学习进度**
已掌握的句子结构类型 / 待加强 / 从未遇到
随着输入句子的增加，进度地图自动丰富

**④ 成就系统**
解锁条件与成就徽章示例：
- 「倒装句终结者」：虚拟语气题连续答对 5 次
- 「定语从句猎手」：支脉识别正确率超过 80%
- 「神韵大师」：L3 专家级题目正确率超过 70%
- 「手术室学徒」：完成首次完整解构
- 「百句斩」：累计解构 100 个句子

**⑤ 分享功能**
成就徽章、学习进度截图可一键生成分享卡片（自然传播）

## 四、技术架构

### 4.1 技术选型

| 层级 | 技术方案 | 选用理由 |
|------|----------|----------|
| 前端框架 | React + Tailwind CSS | 组件化，适合 Vibe Coding 快速迭代 |
| AI 引擎 | Claude API (claude-sonnet-4-5) | 逻辑分析能力强，结构化 JSON 输出稳定 |
| 本地存储 | IndexedDB（via Dexie.js） | 容量大，支持结构化查询，适合存储学习记录 |
| 部署方式 | 静态网页 → Vercel / GitHub Pages | 零成本，开源友好，一键部署 |
| 开源协议 | MIT License | 最大程度开放，知识归用户所有 |

### 4.2 AI 输出协议（System Prompt + JSON 规范）

**System Prompt 核心设定**：

```
Role: 你是一位语言教育专家与逻辑分析师，专注于英语长难句的结构解析与教学。

Task: 接收用户输入的英文句子，按照四步解构算法（信号探测→骨架提取→逻辑分层→语序重排）进行分析，并严格按照以下 JSON 格式输出结果。

Rules:
1. logic_blocks 中所有 text 字段拼接后，必须 100% 等于原句（含空格和标点）
2. JSON 必须合法，不得包含任何 Markdown 标记或多余字符
3. 遇到数学公式，使用 [FORMULA_N] 占位符保护，解析完成后还原
4. 翻译必须遵循：L1 直白平铺，L2 语序重排消除翻译腔，L3 意象脱壳实现信达雅
5. expert_alert 仅在检测到倒装、虚拟语气、插入语、双重否定时触发
```

**JSON 输出规范**：

```json
{
  "skeleton": {
    "subject": "主语文本",
    "verb": "谓语文本",
    "object_complement": "宾语或补语文本",
    "is_inverted": false,
    "inversion_type": ""
  },
  "logic_blocks": [
    {
      "id": 1,
      "text": "原句中的文本片段（含原始空格）",
      "type": "main | context | modifier | nested | signal",
      "level": 0,
      "explanation": "这个成分的功能，用学生能懂的语言说明",
      "referent": "如果是代词，指明其先行词；否则为空"
    }
  ],
  "pattern_matches": [
    {
      "pattern": "Without A, B will...",
      "tip": "条件前置套路：先说缺少什么，再说导致什么后果"
    }
  ],
  "graded_translations": {
    "foundation": "骨架译文：直白平铺，主谓宾齐全",
    "intermediate": "逻辑译文：语序重排，符合中文直觉",
    "professional": "神韵译文：意象脱壳，信达雅"
  },
  "expert_alert": "特殊结构提示，无则为空字符串",
  "confidence": 95
}
```

**新增字段说明**：
- `pattern_matches`：自动识别句子中匹配的万能转化套路
- `referent`：代词指代追溯（it, which, that 的先行词）
- `level`：逻辑块的嵌套深度（0=主句，1=第一层从句，2=嵌套从句）
- `confidence`：AI 对本次解析的置信度（低于 60% 时切换为词块模式）

### 4.3 挑战模式出题协议

**出题 System Prompt 核心逻辑**：

```
基于用户提供的原句和已有解构结果，生成 4 道不同级别的选择题。

每道题包含：
- question_type: L1_skeleton | L2_reorder | L3_imagery | L4_strategy
- question: 题目文本
- options: [A, B, C, D] 四个选项
- correct: 正确选项字母
- rationale: 选择该答案的原理解释（用学生能懂的语言）
```

### 4.4 边界防御机制

| 异常类型 | 检测条件 | 处理方式 |
|----------|----------|----------|
| 倒装句 | Had/Never/Only/Rarely 等前置 | is_inverted: true + expert_alert 触发 |
| 虚拟语气 | would have done / were to 等 | 标注 hypothetical + 提示非真实性 |
| 公式保护 | LaTeX 格式 $...$ 或数字公式 | 占位符 [FORMULA_N] 保护，输出后还原 |
| 代词追溯 | it / which / that / they 等 | referent 字段注明先行词 |
| 极度残缺 | confidence < 60% | 切换为"词块(Chunk)解析模式" |
| 超长嵌套 | 60词以上 / 嵌套超过3层 | level 字段清晰标注，分层展示 |
| 非英文输入 | 检测到中文/乱码 | 返回提示，不进行解析 |

### 4.5 前端渲染逻辑

**流式渲染（减少加载焦虑）**：
按照 logic_blocks 的顺序逐个绘制色块，而不是等待全部 JSON 返回后再渲染。用户会看到句子逐块"亮起来"，像手术刀切开一样。

**动效设计**：
- 色块出现时：fade-in + 轻微上移动效
- 骨架模式切换时：修饰成分淡出，主干高亮放大
- 语法气泡：点击色块后弹出，背景磨砂玻璃效果
- 译文展开：accordion 动效，逐级展开

**视觉规范（Apple Minimalist）**：
- 字体：SF Pro Display（英文）/ PingFang SC（中文）
- 主色：#1D1D1F（Apple Jet Black）
- 辅色：#424245（正文灰）
- 强调色：#0071E3（Apple Blue）
- 背景：#FFFFFF / #F5F5F7（浅灰）
- 材质：磨砂玻璃（backdrop-filter: blur）用于浮层
- 间距：行高 1.6，大量留白，内容密度低
- 配色映射：
  - 主语 → #0071E3（蓝）
  - 谓语 → #34C759（绿）
  - 宾语 → #FF9500（橙）
  - 修饰 → #8E8E93（灰）
  - 连接词 → #AF52DE（紫，下划线）

## 五、AI 引擎质检量表（上线前必须通过）

### 评分维度（满分 100 分）

**① 骨架提取精准度（40分）**
- 40分：在倒装、插入语干扰下依然精准提取 S+V+O
- 扣分：找错主语 -15 分；从句动词误判为主句动词 -10 分

**② 逻辑分层稳定性（20分）**
- 20分：logic_blocks 拼接后与原句 100% 吻合，类型分类无误
- 扣分：拼接丢失空格 -5 分；定语状语不分 -5 分

**③ 翻译去"翻译腔"程度（30分）**
- 30分：Level 3 实现意象脱壳，语序完全符合中文直觉
- 扣分：Level 3 仍保留多层嵌套"的"结构 -10 分；未执行状语前置 -5 分

**④ 边界防御与警示（10分）**
- 10分：准确保护公式，对虚拟语气/倒装给出 Expert Alert
- 扣分：公式损坏 -5 分；异常句式无预警 -5 分

**质检红线（任意一条触发即不合格）**：
- 严禁丢失原句中的空格或标点
- 严禁在祈使句中抓错主语（祈使句主语为隐含的 You）
- JSON 格式必须 100% 合法，无 Markdown 杂质
- logic_blocks 拼接不等于原句，视为严重错误

### MVP 核心测试用例

| 难度 | 测试句子 | 考查点 | 合格标准 |
|------|----------|--------|----------|
| 中等 | The idea that technology always leads to progress is often questioned by historians. | 同位语从句 | 主干：Idea is questioned |
| 高难 | What seems to be a simple task actually requires years of rigorous training. | 主语从句 | 主干：What requires training |
| 极难 | Had the committee known about the risks earlier, the project would have been cancelled. | 虚拟语气倒装 | 触发 expert_alert，主干：Project would have been cancelled |
| 翻译 | Only by working together can we overcome the challenges ahead. | 状语前置重组 | L2 译文：只有通力合作，我们才能克服挑战 |
| 意象 | The silence in the room was palpable. | 意象脱壳 | L3 译文：屋子里的寂静仿佛凝固了，触手可及 |

## 七、Vibe Coding 开发元指令

1. **意图优先**：写任何代码前，先用一句话说清楚「这段代码在验证什么假设」
2. **最短路径**：永远从能跑通的最小版本开始，每一步产出都必须是真实可用的
3. **心理脱钩**：代码随时可以重写，不对任何实现方式产生情感绑定
4. **显性标注**：任何不确定的地方必须明确标出，禁止假装懂了
5. **词义校准**：开始新模块前，先对齐「当我说 X 时，具体画面是什么」
6. **放弃清单**：每个阶段，「不做什么」比「做什么」更重要
